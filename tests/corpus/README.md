# Test Document Corpus

Hermetic test fixtures generated by `tests/corpus/_generate.py`.

## Regenerate

After `uv sync` (which installs `python-docx`, `python-pptx`, `openpyxl`,
`reportlab` as dev deps), run:

```bash
python -m tests.corpus._generate
```

This creates any missing files. Existing files are kept untouched so you
can hand-modify a fixture for a specific test case.

## Inventory

| File              | Pages   | Format | Notes                                |
| ----------------- | ------- | ------ | ------------------------------------ |
| small.docx        | 3       | DOCX   | plain text + page breaks             |
| medium.docx       | 100     | DOCX   | lorem ipsum + page breaks            |
| simple.pptx       | 5       | PPTX   | title + bullet on standard layout    |
| complex.pptx      | 20      | PPTX   | text + shapes (no embedded images)   |
| single_sheet.xlsx | varies  | XLSX   | 100 rows × 20 columns                |
| multi_sheet.xlsx  | varies  | XLSX   | 4 sheets of 10, 20, 30, 40 rows      |
| simple.pdf        | 10      | PDF    | generated by ReportLab               |

PPTX/XLSX page counts depend on Aspose's rendering decisions and are not
asserted at planner level — integration tests use page-count from probe.

## Committing

After `_generate.py` runs, commit the generated files alongside the script.
The script is the source of truth; binaries are checked in for hermetic
test runs that don't require the doc-generation deps.
